# 问题修复记录

## FIX-001 SQLite 测试数据库文件被占用

发现时间：2026年6月25日

问题阶段：自动化测试

问题现象：

运行单元测试时，测试逻辑本身通过，但 `tearDown()` 清理临时目录失败，Windows 提示 SQLite 数据库文件正在被另一个程序使用。

报错摘要：

```text
PermissionError: [WinError 32] 另一个程序正在使用此文件，进程无法访问: test.sqlite3
```

原因分析：

`sqlite3.Connection` 用作上下文管理器时会自动提交或回滚事务，但不会自动关闭连接。原代码中 `with get_connection(...) as connection:` 结束后数据库连接仍可能保持打开状态，导致 Windows 无法删除临时数据库文件。

修复方案：

将 `get_connection()` 改为使用 `contextlib.contextmanager` 封装，确保每次使用后执行：

1. `connection.commit()`
2. `connection.close()`

修复结果：

已重新运行自动化测试，结果通过：

```text
Ran 5 tests
OK
```

该问题已修复。

## FIX-002 Streamlit 热重载后导入旧模块

发现时间：2026年6月25日

问题阶段：页面冒烟测试

问题现象：

在为自测练习新增 `find_unanswered_questions()` 后，自动化测试和语法检查均通过，但浏览器访问 Streamlit 页面时出现导入错误。

报错摘要：

```text
ImportError: cannot import name 'find_unanswered_questions' from 'src.quiz_engine'
```

原因分析：

代码文件已经更新，但后台运行中的 Streamlit 服务仍保留旧的模块加载状态。该问题不是业务逻辑错误，而是开发服务器在热重载过程中没有正确刷新新增函数。

修复方案：

停止原有 Streamlit 进程，重新启动应用服务。

修复结果：

重新访问 `http://localhost:8501/` 后，首页和自测练习页面均可正常打开。继续使用 Playwright CLI 验证自测页面，确认未作答提交会出现提示，不再出现导入错误。

该问题已修复。
