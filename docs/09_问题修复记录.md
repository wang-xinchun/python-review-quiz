# 问题修复记录

## FIX-001 SQLite 测试数据库文件被占用

发现时间：2026年6月25日

问题阶段：自动化测试

问题现象：

运行单元测试时，测试逻辑本身通过，但 `tearDown()` 清理临时目录失败，Windows 提示 SQLite 数据库文件正在被另一个程序使用。

报错摘要：

```text
PermissionError: [WinError 32] 另一个程序正在使用此文件，进程无法访问: test.sqlite3
```

原因分析：

`sqlite3.Connection` 用作上下文管理器时会自动提交或回滚事务，但不会自动关闭连接。原代码中 `with get_connection(...) as connection:` 结束后数据库连接仍可能保持打开状态，导致 Windows 无法删除临时数据库文件。

修复方案：

将 `get_connection()` 改为使用 `contextlib.contextmanager` 封装，确保每次使用后执行：

1. `connection.commit()`
2. `connection.close()`

修复结果：

已重新运行自动化测试，结果通过：

```text
Ran 5 tests
OK
```

该问题已修复。
