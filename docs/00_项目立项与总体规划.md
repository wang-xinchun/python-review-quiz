# Python课程智能复习与自测系统 - 项目立项与总体规划

## 1. 项目名称

Python课程智能复习与自测系统

## 2. 项目定位

本项目面向《Python程序设计》课程期末复习场景，利用课程 PPT、题库、学习记录等数据，构建一个集“课程资料管理、知识点检索、章节自测、错题记录、学习统计、报告生成”于一体的 Python 应用。

项目不仅展示 Python 程序设计能力，也重点展示学生如何规划、调用和评估大模型 Agent 在软件开发过程中的作用。大模型在本项目中不是简单替代编程，而是参与选题论证、需求拆解、模块设计、代码实现、测试设计、问题修复和报告整理。

## 3. 选题理由

### 3.1 与课程内容高度相关

课程 PPT 涉及 Python 基础语法、流程控制、数据结构、字符串、函数、模块、面向对象和文件操作等内容。本项目可以自然覆盖这些知识点，并将课程资料作为项目数据来源，避免选题与课程脱节。

### 3.2 功能规模适合期末大作业

系统既有完整的软件结构，又不会过度复杂。它可以分阶段实现：

1. 基础版本：资料导入、题库维护、自测判分。
2. 完整版本：错题记录、学习统计、报告导出。
3. 提升版本：接入大模型生成题目建议、复习建议和报告摘要。

### 3.3 适合展示大模型 Agent 协作能力

本课程特别考察大模型 Agent 的使用能力。本项目可以清楚呈现：

1. 如何让大模型参与需求分析。
2. 如何让大模型辅助模块拆分和代码设计。
3. 如何通过 Prompt 约束生成结果。
4. 如何验证、修改和记录大模型生成内容。
5. 如何把 AI 协作过程写入最终课程设计报告。

## 4. 用户角色

### 4.1 学生用户

主要使用系统进行课程复习、自测练习、查看错题和学习统计。

### 4.2 管理者用户

主要负责维护课程资料、题库内容和系统数据。对于本课程设计，管理者可以理解为项目开发者本人。

## 5. 核心功能模块

### 5.1 课程资料导入模块

功能目标：

1. 导入课程 PPT/PDF 或整理后的文本资料。
2. 提取章节标题、知识点和正文内容。
3. 将资料保存为结构化数据，供检索和复习使用。

涉及知识点：

1. 文件与目录操作。
2. 字符串处理。
3. 异常处理。
4. 列表、字典等数据结构。

### 5.2 知识点管理模块

功能目标：

1. 按章节浏览知识点。
2. 支持关键词搜索。
3. 支持标记重点或收藏内容。

涉及知识点：

1. 字典和列表。
2. 字符串查找。
3. 数据持久化。

### 5.3 题库管理模块

功能目标：

1. 维护选择题、判断题、填空题等题型。
2. 为题目设置章节、难度、答案和解析。
3. 支持新增、修改、删除和查询题目。

涉及知识点：

1. 面向对象建模。
2. SQLite 或 JSON 数据存储。
3. 函数和模块化设计。

### 5.4 自测练习模块

功能目标：

1. 用户选择章节或题量后开始测试。
2. 系统随机组卷。
3. 用户提交后自动判分。
4. 显示正确答案、解析和得分。

涉及知识点：

1. 分支结构和循环结构。
2. 随机抽题。
3. 函数调用。
4. 数据校验。

### 5.5 错题与学习记录模块

功能目标：

1. 自动记录错题。
2. 保存每次练习的得分、时间、章节和答题结果。
3. 支持错题回顾和重复练习。

涉及知识点：

1. 文件或数据库写入。
2. 时间记录。
3. 数据查询与统计。

### 5.6 学习统计与报告模块

功能目标：

1. 统计不同章节的正确率。
2. 统计错题数量和练习次数。
3. 生成学习分析图表。
4. 导出学习报告。

涉及知识点：

1. 数据统计。
2. 可视化图表。
3. CSV、文本或 Markdown 文件导出。

## 6. 推荐技术路线

### 6.1 首选方案：Streamlit Web 应用

优点：

1. 界面直观，适合展示。
2. 开发效率高。
3. 便于插入图表和表格。
4. 最终报告截图效果较好。

可能使用的库：

1. streamlit：构建交互式 Web 页面。
2. pandas：处理题库和学习记录。
3. sqlite3：保存系统数据。
4. matplotlib 或 plotly：生成统计图表。
5. pypdf 或 pdfplumber：读取 PDF 文本。

### 6.2 备选方案：Tkinter 桌面应用

优点：

1. Python 标准库自带，运行稳定。
2. 适合课程基础要求。
3. 不依赖浏览器和 Web 服务。

缺点：

1. 展示效果不如 Streamlit。
2. 图表和布局实现成本较高。

### 6.3 当前建议

优先采用 Streamlit 方案。如果后续发现环境限制较多，再降级为 Tkinter 方案。

## 8. 项目亮点

1. 数据来源真实：直接围绕本课程 PPT 设计。
2. 功能闭环完整：从资料导入到复习、自测、统计、报告导出。
3. 技术点覆盖全面：基础语法、数据结构、函数、模块、类、文件、数据库、可视化均可体现。
4. 大模型协作过程可追踪：保留需求、Prompt、测试和修复记录。
5. 报告可写性强：每个模块都有明确设计、实现和测试内容。
