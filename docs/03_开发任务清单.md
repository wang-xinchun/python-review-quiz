# 开发任务清单

## 1. 当前状态

已完成：

1. 阅读工作区文件结构。
2. 确认课程 PPT 涵盖 Python 基础、流程控制、数据结构、函数、模块、面向对象和文件操作。
3. 阅读案例报告结构，确认最终报告需要包含摘要、引言、开发工具、需求设计、模块实现、测试和总结等部分。
4. 确定项目主题为“Python课程智能复习与自测系统”。
5. 建立初步项目规划和大模型协作方案。

## 2. 待确认决策

### 2.1 界面技术

推荐选择：

1. Streamlit：展示效果好，适合图表和报告截图。
2. Tkinter：依赖少，偏传统桌面程序。

当前建议：选择 Streamlit。

### 2.2 数据存储

推荐选择：

1. SQLite：适合题库、练习记录和错题数据。
2. JSON/CSV：适合课程资料和导出报告。

当前建议：SQLite + JSON/CSV。

### 2.3 第一版功能范围

第一版应控制范围，保证先跑通核心闭环：

1. 题库管理。
2. 自测练习。
3. 自动判分。
4. 错题记录。
5. 简单统计。
6. 学习报告导出。

课程资料导入、在线大模型 API、多用户登录等功能暂不放入第一版，避免一开始被资料格式、网络环境或权限逻辑拖慢。第一版重点展示 Python 基础知识的综合应用。

## 3. 第一阶段：需求与数据设计

目标：明确系统怎么存数据、有哪些页面、每个模块输入输出是什么。

任务：

1. 确定最终技术路线。
2. 设计数据库表。
3. 设计题目数据格式。
4. 设计练习记录格式。
5. 设计错题记录格式。
6. 设计 Streamlit 页面结构。

产出：

1. `docs/04_需求分析.md`
2. `docs/05_数据库设计.md`
3. 数据库初始化脚本或模块。

## 4. 第二阶段：最小可运行版本

目标：实现系统核心闭环。

任务：

1. 创建项目目录。
2. 创建 `requirements.txt`。
3. 实现数据库初始化。
4. 实现题库新增和查询。
5. 实现按章节随机抽题。
6. 实现答题提交和自动判分。
7. 实现错题保存。
8. 实现基础统计。
9. 运行应用并截图。

验收标准：

1. 能启动应用。
2. 能新增题目。
3. 能开始一次测试。
4. 能提交答案并得到分数。
5. 答错的题目能被记录。
6. 能看到基础统计结果。
7. 能导出一份简单学习报告。

## 5. 第三阶段：课程资料与题库增强

目标：让系统更贴合 Python 课程内容。

任务：

1. 从课程 PPT 中整理章节结构。
2. 建立初始题库。
3. 使用大模型辅助生成题目候选。
4. 人工检查题目正确性。
5. 导入题库数据。
6. 增加知识点搜索功能。

验收标准：

1. 题库覆盖主要章节。
2. 每章至少有若干题目。
3. 题目包含答案和解析。
4. 知识点或题目支持按关键词搜索。

## 6. 第四阶段：统计与报告导出

目标：提升展示效果和报告可写性。

任务：

1. 统计练习次数。
2. 统计平均正确率。
3. 统计章节正确率。
4. 统计错题数量。
5. 绘制图表。
6. 导出学习报告。

验收标准：

1. 图表显示正常。
2. 数据和练习记录一致。
3. 可生成文本或 Markdown 学习报告。
4. 报告中包含薄弱章节和复习建议。

## 7. 第五阶段：测试与修复

目标：保证系统稳定，积累最终报告素材。

任务：

1. 设计功能测试用例。
2. 测试题库为空的情况。
3. 测试中文路径和中文内容。
4. 测试答案为空或格式错误。
5. 测试重复提交。
6. 测试统计数据是否准确。
7. 记录问题和修复过程。

产出：

1. `docs/06_测试记录.md`
2. `docs/07_问题修复记录.md`

## 8. 第六阶段：最终报告整理

目标：形成符合课程要求的报告。

任务：

1. 整理项目背景和意义。
2. 整理开发工具介绍。
3. 整理需求分析。
4. 整理系统设计。
5. 整理模块实现。
6. 整理测试记录。
7. 整理大模型 Agent 协作过程。
8. 插入系统截图。
9. 生成最终 Word 报告。

产出：

1. 课程设计报告初稿。
2. 最终 Word 版报告。
3. 系统源码和运行说明。

## 9. 大模型协作任务清单

后续每个开发阶段都应保留至少一条 AI 协作记录。

建议记录：

1. 选题规划记录。
2. 需求分析记录。
3. 数据库设计记录。
4. 题库生成记录。
5. 代码修复记录。
6. 测试用例设计记录。
7. 报告整理记录。

## 10. 下一步建议

下一步应完成两个决策：

1. 确认界面技术采用 Streamlit。
2. 确认第一版先做“题库 - 自测 - 判分 - 错题 - 统计”闭环。

确认后即可开始创建代码目录和数据库设计文档。

范围控制补充：

1. 第一版不做复杂登录系统。
2. 第一版不接入实时大模型 API。
3. PDF 资料导入作为增强功能，不影响核心闭环验收。
4. 题库初始数据优先手工整理并经人工检查，保证内容正确。
