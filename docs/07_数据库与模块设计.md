# 数据库与模块设计

## 1. 设计目标

本项目第一版使用 SQLite 保存题库、练习记录和答题记录。数据库设计保持轻量，重点服务于以下功能：

1. 题库浏览、搜索和新增。
2. 自测练习和自动判分。
3. 错题记录。
4. 学习统计。
5. 学习报告导出。

## 2. 数据库文件

默认数据库文件：

```text
data/review_quiz.sqlite3
```

数据库文件为运行时数据，不提交到公共 GitHub 仓库。

## 3. 数据表设计

### 3.1 questions 题目表

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | INTEGER PRIMARY KEY | 题目编号 |
| chapter | TEXT | 所属章节 |
| question_type | TEXT | 题型，包含 single、judge、blank |
| difficulty | TEXT | 难度 |
| stem | TEXT | 题干 |
| options | TEXT | JSON 格式选项 |
| answer | TEXT | 正确答案 |
| explanation | TEXT | 解析 |

### 3.2 quiz_records 练习记录表

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | INTEGER PRIMARY KEY | 练习记录编号 |
| quiz_time | TEXT | 测试时间 |
| chapter_scope | TEXT | 测试章节范围 |
| score | INTEGER | 得分 |
| total | INTEGER | 总题数 |
| correct_count | INTEGER | 正确题数 |

### 3.3 answer_records 答题记录表

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | INTEGER PRIMARY KEY | 答题记录编号 |
| quiz_id | INTEGER | 所属练习记录 |
| question_id | INTEGER | 题目编号 |
| user_answer | TEXT | 用户答案 |
| correct_answer | TEXT | 正确答案 |
| is_correct | INTEGER | 是否正确，1 表示正确，0 表示错误 |
| created_at | TEXT | 创建时间 |

错题本不单独建表，而是从 `answer_records` 中筛选 `is_correct = 0` 的记录得到。这样可以避免重复保存数据。

## 4. 模块设计

### 4.1 src/models.py

职责：

1. 定义题目、练习记录等数据类。
2. 实现题目答案检查方法。
3. 保存题型常量。

主要类：

1. `Question`
2. `StudyRecord`

### 4.2 src/question_bank.py

职责：

1. 初始化数据库。
2. 读取和写入题目。
3. 按章节、题型、关键词查询题目。
4. 导入初始题库。

### 4.3 src/quiz_engine.py

职责：

1. 按章节随机抽题。
2. 接收用户答案。
3. 自动判分。
4. 返回答题明细。

主要类：

1. `QuizSession`

### 4.4 src/record_manager.py

职责：

1. 保存练习记录。
2. 保存每道题答题结果。
3. 查询错题记录。
4. 查询历史练习记录。

### 4.5 src/statistics_service.py

职责：

1. 统计题目数量。
2. 统计练习次数。
3. 统计平均正确率。
4. 统计章节正确率。
5. 统计错题分布。

### 4.6 src/report_exporter.py

职责：

1. 根据统计结果生成 Markdown 学习报告。
2. 将报告导出到 `exports/` 目录。
3. 返回报告文本，供界面下载。

### 4.7 app.py

职责：

1. 构建 Streamlit 界面。
2. 调用各业务模块。
3. 提供首页、题库、自测、错题、统计和报告页面。
