## 3. 系统需求分析与设计

### 3.1 需求分析

功能需求：

1. 课程资料导入与查看。
2. 知识点搜索。
3. 题库管理。
4. 自测练习。
5. 错题记录。
6. 学习统计。
7. 报告导出。

非功能需求：

1. 界面简洁。
2. 数据保存稳定。
3. 中文路径和中文内容处理正常。
4. 操作流程清晰。
5. 功能模块便于维护。

### 3.2 系统角色分析

学生用户：

1. 查看知识点。
2. 进行自测。
3. 查看错题和统计结果。

管理者用户：

1. 导入课程资料。
2. 维护题库。
3. 查看学习数据。

可设计以下数据表或数据文件：

| 字段 | 含义 |

| difficulty | 难度 |

| 字段 | 含义 |
