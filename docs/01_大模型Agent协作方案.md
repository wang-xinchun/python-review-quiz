# 大模型 Agent 协作方案

## 1. 协作目标

本项目将大模型 Agent 作为软件开发协作伙伴，用于辅助完成项目规划、需求分析、代码实现、测试验证和报告整理。使用大模型的重点不是追求“自动生成全部代码”，而是体现学生能够有计划、有约束、有验证地使用 AI 工具完成课程设计。

## 2. 协作原则

### 2.1 人主导，AI 辅助

学生负责确定项目方向、判断功能取舍、确认最终代码和报告内容。大模型负责提供方案、生成初稿、检查问题和辅助实现。

### 2.2 过程可记录

重要协作过程需要保留记录，包括：

1. 需求讨论。
2. 方案比较。
3. Prompt 设计。
4. 代码生成和修改。
5. 测试设计。
6. 错误分析与修复。

### 2.3 结果可验证

大模型生成的代码、文档和测试建议都需要经过人工确认和实际运行验证。报告中应体现“生成 - 检查 - 修改 - 验证”的闭环。

### 2.4 不照抄案例

示例报告只作为格式参考，不复制其选题、内容和表述。本项目采用独立选题，并围绕 Python 课程资料完成设计。

## 3. 大模型参与阶段

### 3.1 选题规划阶段

目标：

1. 分析课程内容和案例报告结构。
2. 比较多个可选项目。
3. 确定最终选题。

典型 Prompt：

```text
请根据 Python 程序设计课程期末大作业要求，帮我比较 3 个适合的项目选题。
要求考虑课程知识点覆盖、实现难度、展示效果、报告可写性和大模型协作体现方式。
```

预期产出：

1. 选题比较表。
2. 最终选题理由。
3. 项目总体规划。

### 3.2 需求分析阶段

目标：

1. 明确用户角色。
2. 拆分功能需求。
3. 确定非功能需求。
4. 形成模块边界。

典型 Prompt：

```text
请作为软件工程助教，帮助我为“Python课程智能复习与自测系统”编写需求分析。
需要包括用户角色、核心功能、输入输出、异常情况和可测试标准。
```

预期产出：

1. 功能需求清单。
2. 模块关系说明。
3. 测试点初稿。

### 3.3 系统设计阶段

目标：

1. 设计项目目录结构。
2. 设计主要类和函数。
3. 设计数据库或数据文件结构。
4. 设计界面页面结构。

典型 Prompt：

```text
请为该系统设计模块结构和数据结构。
要求使用 Python，优先考虑 Streamlit、SQLite 和模块化代码。
请说明每个模块的职责、主要函数和数据流。
```

预期产出：

1. 系统架构说明。
2. 数据表设计。
3. 类和函数说明。
4. 页面流程设计。

### 3.4 编码实现阶段

目标：

1. 按模块逐步实现功能。
2. 发现并修复运行错误。
3. 保持代码可读性和可维护性。

典型 Prompt：

```text
请根据已有项目结构，实现题库管理模块。
要求包括新增题目、查询题目、按章节筛选、删除题目等功能。
代码需要有清晰函数划分，并便于后续测试。
```

预期产出：

1. 可运行代码。
2. 模块说明。
3. 错误修复记录。

### 3.5 测试验证阶段

目标：

1. 设计测试用例。
2. 验证模块功能。
3. 记录异常和修复结果。

典型 Prompt：

```text
请为自测练习模块设计测试用例。
测试内容包括正常答题、空题库、答案为空、重复提交、错题记录和统计结果。
请用表格形式输出测试编号、测试输入、预期结果和实际结果。
```

预期产出：

1. 测试用例表。
2. 测试记录。
3. 问题修复记录。

### 3.6 报告整理阶段

目标：

1. 按课程设计报告格式整理内容。
2. 截取系统运行界面。
3. 总结大模型协作过程。
4. 形成最终报告。

典型 Prompt：

```text
请根据本项目的功能、代码结构、测试记录和大模型协作记录，帮我整理课程设计报告初稿。
报告需要包括摘要、引言、开发工具、需求分析、系统设计、功能实现、测试、大模型协作过程和总结。
```

预期产出：

1. 报告初稿。
2. 图表和截图清单。
3. 参考文献建议。

## 4. Prompt 记录模板

后续开发过程中，可以按以下格式记录关键 Prompt：

```text
使用目标：
输入 Prompt：
大模型输出摘要：
采纳内容：
人工修改：
验证结果：
反思：
```

## 5. AI 协作记录示例

```text
记录编号：AI-001
使用时间：2026年6月
开发阶段：选题规划
使用目标：确定适合 Python 课程期末大作业的项目主题
输入 Prompt：请根据课程内容和报告要求，比较多个项目选题，并说明哪个更适合展示 Python 编程能力和大模型规划能力。
大模型输出摘要：推荐“Python课程智能复习与自测系统”，理由是贴合课程内容、功能闭环完整、易于展示大模型协作过程。
采纳内容：采纳项目主题和核心模块划分。
人工修改：将功能范围控制在课程资料导入、题库、自测、错题和统计五个主要方向。
验证结果：该选题与课程 PPT 内容匹配，报告结构可完整展开。
反思：大模型适合用于选题比较，但最终选择需要结合课程要求和开发时间判断。
```

## 6. 报告中可体现的 AI 应用能力

### 6.1 规划能力

报告中说明如何利用大模型进行选题比较、功能拆解和开发阶段安排。

### 6.2 提问能力

报告中展示结构化 Prompt，不使用简单模糊的“帮我写代码”式指令。

### 6.3 判断能力

报告中说明哪些建议被采纳，哪些建议被修改或放弃，体现人工判断。

### 6.4 验证能力

报告中展示测试记录，说明大模型输出经过运行检查和结果验证。

### 6.5 迭代能力

报告中呈现从初稿到修复再到稳定版本的过程，体现持续改进。

## 7. 注意事项

1. 不应把大模型描述成完全替代开发者。
2. 不应直接复制大模型输出而不检查。
3. 不应让报告显得空泛，要结合具体模块、具体错误和具体测试。
4. 需要保留关键协作记录，作为最终报告素材。
5. 对于 AI 生成题目和解析，应进行人工筛选，避免知识点错误。
